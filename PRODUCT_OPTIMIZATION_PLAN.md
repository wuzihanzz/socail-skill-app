# 产品优化方向

## 当前判断

这个 app 已经从“AI 聊天 demo”进入了“关系练习产品雏形”的阶段。现在最有价值的部分不是模型能不能聊天，而是用户是否能在一次次对话里感到：我正在更好地理解一个人，也正在学会更好地回应别人。

当前已经成立的基础：

- 服务端代理 DeepSeek，避免前端暴露 key，并加入来源限制和限流。
- 三个角色具备独立关系状态、信任度、情绪和长期 memory wing。
- 首页、选择角色、聊天页、档案页已经统一到更克制的视觉方向。
- 用户可以通过对话逐步解锁角色信息，形成“慢慢认识人”的结构。

当前最需要加强的部分：

- 聊天启发还偏模板，不够贴合当前对话。
- 冲突、误会、修复这些真实关系练习还不够系统。
- 信任度只是数值，还没有形成明确的探索激励。
- memory 已经能存，但还需要更细地控制“何时记、何时提、如何提”。
- 首页需要更清楚地告诉用户“今天该练什么”，而不是只是进入聊天。

## 产品原则

### 1. 让用户练习关系，而不是完成任务

这个产品的核心不是通关、刷分或哄角色开心，而是在低风险环境里练习真实对话能力。所有功能都应该服务于“更好地理解、回应、修复、建立信任”。

### 2. 角色要像人，不像教学 NPC

角色不应该频繁解释系统规则，也不应该机械地复述用户记忆。角色可以记得事情，但要像真实的人一样，在合适的情绪和语境里自然提起。

### 3. 教学要藏在体验里

用户需要帮助，但帮助不应该像课程弹窗。聊天启发、关系反馈、档案解锁、冲突修复，都应该尽量贴近当前对话发生。

### 4. 信息逐步显露

职业、年龄、MBTI、真实姓名、深层背景等信息不应该在选择前直接暴露。用户要通过对话、信任和探索逐渐获得这些内容。

### 5. 系统能力不外显

长期记忆、memory wing、独立记忆、模型配置等是产品底层能力，不需要作为用户可见卖点反复展示。用户只需要感受到“这个人记得我”和“关系在变化”。

## 优先级路线

## P0：聊天启发动态化

目标：让“可以这样接”从静态模板变成当前对话里的真实辅助。

现状问题：

- 当前启发主要根据最后一句话粗略生成，场景适配弱。
- 缺少对用户当前意图的判断，比如安慰、追问、道歉、转移话题、边界表达。
- 冲突或冷场时，启发不够有修复方向。

建议设计：

- 建立 `ConversationIntent`：识别当前场景属于日常寒暄、情绪倾诉、压力表达、冲突、拒绝、冷场、亲近、信息探索等。
- 建立 `ReplyMove`：每个场景给出 2-3 个可选回应动作，比如“共情一句”“轻轻追问”“承认影响”“给出边界”“换个更温和的问法”。
- 启发文案不要太长，像真实用户会发出去的话，而不是教学解释。
- 对话状态紧张时，优先给修复型回复；状态良好时，优先给探索型回复。

验收标准：

- 同一套启发不会在不同场景里机械重复。
- 用户刚惹对方不高兴时，启发会优先建议修复。
- 用户面对对方倾诉时，启发会优先建议共情，而不是立刻给建议。

## P1：冲突与修复系统

目标：让关系不只会线性增长，也会因为误解、冷淡、冒犯而出现波动，并给用户练习修复的机会。

现状问题：

- 负面消息会降信任，但后续修复路径不清楚。
- 角色的 upset 状态不够持续，用户难以感到“关系真的被影响了”。
- 冲突没有分层，轻微冷场和严重冒犯容易混在一起。

建议设计：

- 增加 `conflictState`：
  - `none`：无冲突
  - `awkward`：冷场或轻微不适
  - `hurt`：对方受伤
  - `withdrawn`：对方开始退缩
  - `repaired`：用户完成修复
- 增加 `repairOpportunity`：当关系受损时，系统提示模型保持情绪延续，并观察用户是否道歉、解释、承认影响、改变表达。
- 信任度变化不只看一句话，也看连续 2-3 轮是否在修复。
- 修复成功后不要立刻“满血恢复”，而是让角色逐渐松动。

验收标准：

- 用户说伤人的话后，角色不会下一轮就完全忘记。
- 用户真诚道歉时，角色会有明显但克制的缓和。
- 聊天启发会根据冲突状态给出修复建议。

## P2：信任 milestone 与探索激励

目标：让信任度从纯数字变成用户愿意继续探索的关系进度。

现状问题：

- 信任度有数值，但用户不清楚进度意味着什么。
- 已了解特质有展示，但缺少“下一步要探索什么”的动力。
- 档案页更像资料页，还没有形成成长反馈。

建议设计：

- 设计 4 个关系阶段：
  - `初识`：只知道昵称和表层状态
  - `熟悉`：开始了解日常压力和偏好
  - `信任`：对方愿意讲脆弱部分
  - `亲近`：对方会主动延续过去话题
- 每个阶段有 1-2 个 milestone，例如“第一次认真回应对方压力”“第一次完成冲突修复”“第一次被对方主动提起过去的事”。
- 档案页增加“下一步可以探索”的轻提示，但不剧透具体个人信息。
- 首页可以显示“今天适合练习：修复 / 倾听 / 追问 / 边界”。

验收标准：

- 用户能理解信任度变化带来的实际差异。
- 档案页能激励继续聊，而不是只展示已知信息。
- milestone 不会变成游戏化噪音，仍保持克制。

## P3：Memory Palace 深化

目标：让长期记忆成为角色生命感的来源，而不是生硬召回。

现状问题：

- memory wing 已经有结构，但记忆重要性、召回时机和引用方式还可以更细。
- 用户事实、角色事实、关系事件、冲突事件需要更明确地区分。
- 角色主动提起记忆时需要更克制。

建议设计：

- 将 drawer 分为更明确的类型：
  - `user_fact`：用户自己的信息
  - `shared_event`：双方共同经历的对话事件
  - `character_reaction`：角色对某事的情绪反应
  - `conflict`：未解决或已修复的冲突
  - `preference`：用户表达过的偏好和边界
- 增加 `recallPolicy`：
  - 用户明确问“还记得吗”时可以直接召回。
  - 情绪或主题高度相关时可以自然带一句。
  - 日常闲聊时不要无端提记忆。
  - 冲突记忆只在修复、重复冒犯、关系状态相关时出现。
- 增加记忆衰减或压缩：低重要度记忆合并进 summary，避免 localStorage 无限膨胀。

验收标准：

- 角色不会没事就说“我记得你之前说过”。
- 当用户追问过去信息时，角色能稳定召回关键内容。
- 冲突修复过的记忆不会永远以负面方式影响关系。

## P4：首页与日常练习流

目标：首页不只是入口，而是帮助用户开始一次合适的练习。

现状问题：

- 首页已经变得统一，但“今天该做什么”的行动导向还可以更强。
- 今日练习目前是静态 tips，和角色状态没有强关联。

建议设计：

- 首页主卡片根据当前关系状态推荐行动：
  - 没开始：选择一个对象，练习自然开场。
  - 有冷场：回到上一段对话，试着温和接住。
  - 有冲突：先做一次修复。
  - 关系良好：探索一个新话题。
- 今日练习从静态 tips 改为根据最近对话状态推荐。
- 保留简洁，不增加复杂任务面板。

验收标准：

- 用户打开首页后能立刻知道下一步做什么。
- 首页推荐与真实对话状态相关。
- 不暴露系统配置和角色未解锁信息。

## P5：语音输入

目标：降低输入成本，让练习更像真实表达。

建议设计：

- 第一版只做语音转文字，不做语音回复。
- 输入框旁增加麦克风按钮。
- 长按或点击开始录音，结束后自动填入文本，用户确认后发送。
- 保留文本编辑能力，避免误识别直接发出。

技术方向：

- 浏览器端优先使用 Web Speech API 做 MVP。
- 如果兼容性不够，再考虑服务端语音识别。
- 语音原始音频不默认保存，只保存最终文本。

验收标准：

- 用户可以用语音填入输入框。
- 识别结果不会未经确认直接发送。
- 权限失败时有清楚但不打扰的 fallback。

## 推荐实施顺序

### 第一阶段：把核心聊天练好

1. 动态聊天启发
2. 冲突状态与修复路径
3. memory recall policy 微调

原因：这三项直接决定用户是否觉得“这个对话值得继续”。

### 第二阶段：把关系进度做成动力

1. 信任阶段命名
2. milestone 设计
3. 档案页“下一步探索”提示

原因：当对话体验成立后，需要给用户一个继续回来的理由。

### 第三阶段：降低使用门槛

1. 首页日常练习流
2. 语音输入
3. 移动端细节优化

原因：这阶段服务留存和使用频率。

## 不建议现在做的事

- 不急着增加大量新角色。当前角色还没有被充分打磨，先把一个角色聊得真实更重要。
- 不急着做账号和云同步。现阶段产品体验还在成型，账户系统会增加复杂度。
- 不急着做社区和排行榜。这个产品更适合私密练习，公开比较可能破坏氛围。
- 不急着展示模型、记忆、配置能力。这些应该留在系统内部，让用户感受到结果即可。

## 成功指标

短期可以先用定性指标：

- 用户是否愿意连续聊 10 轮以上。
- 用户是否会主动回到同一个角色。
- 用户是否能感知角色记得过去的事，但不觉得突兀。
- 用户在冲突后是否知道如何修复。
- 用户是否觉得启发按钮“刚好能帮我接一句”。

后续如果有埋点，可以看：

- 平均会话轮数
- 次日回访率
- 启发按钮点击率
- 冲突后的修复成功率
- milestone 达成率

## 一句话方向

把它做成一个安静、真实、能长期陪用户练习关系能力的对话空间，而不是一个炫技的 AI 聊天页面。
